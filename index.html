Addendum

In converting my resume to HTML, I structured the content using semantic tags for better readability and accessibility. The header contains my name and contact details, followed by different sections like Objective, Education, Skills, Internships, Projects, and Extracurricular Activities. Each section is enclosed within <section> tags, and lists are formatted using <ul> and <li> elements. I used <a> tags for email and LinkedIn links, ensuring navigation ease. The structure maintains clarity while being machine-readable for search engines and assistive technologies.

Stylistic Choices

  • Font Choices: I chose Arial as the primary font for its readability and modern appearance. It is a sans-serif font that provides a clean look.
  • Heading Sizes: The main heading (h1) is set to 2.5em to make it prominent, while h2 and h3 are sized at 2em and 1.5em respectively to create a clear hierarchy.
  • Color Choices: I used a light background color (#f4f4f4) for the body to reduce strain on the eyes, with dark text for contrast. Headings are colored in shades of blue and gray to differentiate them from the body text.
  • Margin and Padding: I applied consistent margins and padding to create space between elements, enhancing readability. The article has padding for a clean look, while headings have top margins to separate them from preceding content.

HTML Abuse

In my original HTML, I used an <h2> tag for the title of my essay because it visually looked better. I have corrected this to use an <h1> tag, as it is the highest-level heading on the page. This change improves the semantic structure of the document.
